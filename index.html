NBA Player Analytics & Market Intelligence App

Problem

Sports bettors and fantasy players have no single place to cross-reference a player's recent performance trends with live prop line movement across sportsbooks. Getting a complete picture requires multiple tabs, multiple sites, and no real way to connect the data.

Solution

A full stack mobile app that pulls live NBA player stats, visualizes performance trends across the last 30 to 40 games, and surfaces real-time prop odds from DraftKings, FanDuel, and Bovada all in one place. A background worker keeps lines fresh every 60 seconds and alerts users when movement hits a defined threshold.

Key Features

  • Searchable player database with interactive stat charts and filters for stat type, game range, location, opponent, rest, and game margin
  • Live market screen comparing prop odds across DraftKings, FanDuel, Caesars, ESPNBet, and Bovada with implied probability and expected value
  • Line movement tracker showing opening line vs current line with directional indicators
  • Background worker refreshing lines every 60 seconds with updates stored in PostgreSQL
  • Tracked players tab showing active alerts with sportsbook and line details
  • LLM-powered AI Analysis tab for natural language stat queries
  • Game matchup screen showing likely starters per team pulled from home screen cards

Challenges

  • Normalizing data across multiple external APIs into a consistent internal schema
  • Managing API rate limits on the BallDontLie free tier during playoffs when game volume spikes
  • Building a caching layer to serve prop data without hammering external endpoints on every request

Future Goals

  • Deploy the app publicly so anyone can access it without running it locally
  • Add push notifications for line movement alerts so users don't have to have the app open
  • Expand the AI Analysis tab to support more complex multi-player comparison queries
  • Integrate injury reports and player news so market context is available alongside the odds
  • Add historical line movement data so users can see how lines have moved over the full day
  • Evolve the AI Analysis tab into a full AI agent that autonomously monitors player trends, detects sharp line movement, and surfaces actionable alerts without the user needing to ask

AI News Bias & Misinformation Detector

Problem

Most people can't easily tell whether the news they're reading is politically slanted or using emotionally charged language to push a narrative. Tools that do this exist but they rate entire outlets rather than individual articles and none are automated in real time.

Solution

A web app where you paste any article URL or text and get back a structured analysis in seconds. Political bias rating, sentiment score, misinformation risk, and a breakdown of loaded language. Built a multi-layer pipeline combining an LLM classification layer with a locally-run HuggingFace transformer and a rule-based distortion analyzer so the system produces independent signals rather than relying on a single model call.

Key Features

  • Paste any URL and the scraper pulls article text automatically via BeautifulSoup
  • Political bias scored on a -5 to +5 scale with plain English context for each rating
  • Dual sentiment analysis using Groq/Llama and HuggingFace RoBERTa blended at 50/50
  • Rule-based distortion analyzer flags loaded words, emotional punctuation, and caps ratio independent of the AI
  • Blended risk score combining AI misinformation output and rule-based distortion at 60/40 weighting
  • Article comparison engine measuring narrative divergence between two sources on the same topic

Challenges

  • Avoiding a pure AI wrapper by building independent rule-based scoring that runs without any model call
  • HuggingFace memory limits on the free deployment tier required a graceful fallback when torch is unavailable
  • Getting consistent structured JSON output from the LLM across varied article formats and lengths

Future Goals

  • Fine-tune a Hugging Face classifier on a labeled bias dataset to replace or supplement the LLM call
  • Build a dashboard showing average bias scores across sources analyzed over time
  • Add a side-by-side comparison view for two articles on the same topic
  • Scale the dataset to 500 or more labeled articles to support model training and evaluation
  • Add a NewsAPI integration to automatically pull and analyze articles on a schedule
  • Evolve the system into a fully autonomous AI agent that monitors news sources on its own, flags high-risk content, and delivers structured reports without any user input

SEO Content Prioritization Engine

Problem

SEO teams spend hours manually reviewing pages to decide what to fix first. There is no systematic way to weigh internal link structure, keyword coverage, and page importance together. Most decisions come down to gut feel or whoever is loudest in the room.

Solution

A Python system that models a website as a directed graph, scores every page by structural importance and keyword relevance, and uses a priority engine to surface the highest impact pages to improve first. Still in early development with the core data structures being built out and tested on sample data before moving to real site inputs.

What's Built So Far

  • Basic directed graph structure where pages are nodes and internal links are edges
  • Early implementation of a PageRank style scoring approach that is still being refined
  • Keyword indexing using hash maps to track which pages cover which terms
  • Experimenting with a heap based approach to rank pages by priority

What's Left

  • Finishing and validating the PageRank scoring so importance scores are accurate across different graph structures
  • Building out the priority engine fully and tuning the weighting between link structure and keyword relevance
  • Web crawler to ingest real websites automatically instead of manually structured sample data
  • Google Search Console integration to use real search impression and click data instead of manually tagged keywords
  • Output layer to generate a usable report rather than just printing to terminal
  • Testing against real sites to validate the scoring produces meaningful recommendations

Challenges

  • Balancing multiple signals into a single priority score without any one factor dominating the result
  • Handling circular link structures in the graph without importance scores diverging during iteration
  • Defining what a meaningful content gap actually is versus a page that just has not been optimized yet

Future Goals

  • Build a web crawler to ingest real websites instead of manually structured JSON input
  • Integrate Google Search Console data to incorporate real search impression and click data into scoring
  • Add competitor gap analysis by crawling competing sites and comparing keyword coverage side by side
  • Build a Flask web interface so non technical users can run audits without touching the command line
  • Generate a full audit report as a PDF with prioritized recommendations and estimated impact per page
  • Add semantic keyword clustering using embeddings so pages targeting similar topics get grouped automatically
  • Evolve into a scheduled agent that re crawls the site weekly and tracks improvement over time
  • Support multi site analysis so agencies can run audits across multiple client domains from one tool
